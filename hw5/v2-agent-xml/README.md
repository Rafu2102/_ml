# AI Agent 檔案存取安全防護系統

本專案實作了一個具有**最高安全等級、防範路徑越界存取**的安全防護 AI 代理人：[Agent_Secure.py](./Agent_Secure.py)。

本系統在原版 `agent0.py` 基礎上，整合了強大的靜態分析安全攔截器（Interceptor），能夠百分之百攔截 AI 代理人企圖對專案目錄外部進行的任何檔案讀寫、刪除或敏感工具執行操作，並提報人工審查（Human-in-the-Loop）。

---

## 安全防護機制設計

為確保防護機制「零漏洞」，系統採用了以下防禦策略：

### 1. 工作區限縮（Workspace Localization）
* 原版程式會將日誌與記憶儲存在系統全域的 `~/.agent0`。安全版將工作目錄重新設定在專案目錄內部的 `.agent0` 資料夾，避免對系統其他區域的存取。

### 2. 嚴格的分詞與運算子拆分
* 為防範透過 Pipeline（管線）或重新導向運算子拼接路徑（例如：`echo text > ../out.txt`），系統會先使用正則表達式將所有的運算子（`&&`, `||`, `;`, `|`, `&`, `>`, `<`, `` ` ``, `$`, `(`, `)`）替換為空格，再使用 `shlex.split` 做 Windows 親和的分詞，使任何潛在的路徑單詞浮現，避免被 shell 語法混淆。

### 3. 動態變數與編碼混淆阻斷
* 系統會先提取指令中的所有變數（如 `%VAR%` 或 `$VAR`），若在系統環境變數中找不到該變數的定義，將直接攔截。這能完全防範於執行期動態賦值進行混淆繞過的攻擊手法。

### 4. 絕對路徑越界檢查（防止相鄰目錄漏洞）
* 每個 Token 都會解析為絕對真實路徑（`os.path.realpath`），並考慮 Windows 的大小寫不敏感性。
* 採用嚴密的 `is_inside` 比對演算法，僅允許在專案目錄本身及子目錄內的操作，能有效防範如 `v2-agent-xml-alt` 這類利用同名開頭資料夾進行的越界繞過。

### 5. 敏感工具與解譯器黑名單
* 嚴格限制 `python`、`node`、`powershell`、`base64`、`pip`、`npm`、`curl` 等可能被用來下載或執行任意程式碼的工具。未經使用者核可，一律禁止執行。

## 如何執行

請在終端機中切換至本目錄，並執行以下指令啟動安全代理人：

```powershell
python Agent_Secure.py
```

當代理人企圖執行不安全指令時，系統將會輸出警告並暫停：

```text
⚠️  [安全警告] 偵測到指令企圖存取外部路徑或執行限制的敏感工具！
原因：企圖存取外部路徑：'C:\Windows'
指令：dir C:\Windows
是否核可此操作？(y/n): 
```
* 輸入 `y` 或 `yes`：允許執行該指令。
* 輸入 `n` 或其他任意鍵：攔截並拒絕該指令。
